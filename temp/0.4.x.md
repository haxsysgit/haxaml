# Haxaml 0.4.x Roadmap

Theme for the 0.4 series:

`reconcile -> adoption -> docs -> verify depth -> context efficiency -> stability`

## 0.4.1: Reconcile Polish + Better Fix Suggestions

Goal:
Make `haxaml_reconcile` genuinely useful for agents and humans.

Focus:

- Improve conflict messages.
- Improve `human_summary`.
- Add clearer `fix_actions`.
- Add examples of common conflicts.
- Make output easier for agents to repair from.
- Add more tests around edge cases.

Success outcome:
Agents can run reconcile, understand exactly what is wrong, and know what safe fix to apply.

Possible changes:

- Add `fix_confidence`: `low | medium | high`
- Add `safe_to_auto_apply`: `false` for now
- Add `related_files`
- Add `why_it_matters`
- Add `suggested_next_tool`

Example:
`haxaml_reconcile` should not just say "map/rules conflict."
It should say:
"map.yaml is canonical for module paths. rules.yaml repeats a stale path. Update rules.yaml to reference the module name instead of redefining the path."

## 0.4.2: Adopt Plan Improvements

Goal:
Make Haxaml easier to introduce into existing AI-agent projects.

Focus:

- Improve detection of existing agent files:
  - `AGENTS.md`
  - `CLAUDE.md`
  - `copilot-instructions.md`
  - `.cursor/rules`
  - `.windsurf/rules`
  - README agent sections
- Add better inventory summaries.
- Add clearer migration steps.
- Detect duplicated or conflicting instructions.
- Suggest where each type of content belongs in FRAME.

Success outcome:
A project with messy AI instructions can run `haxaml_adopt_plan` and get a clear, non-destructive path into FRAME.

Important rule:
Still do not auto-write or overwrite existing files by default.

## 0.4.3: MCP Docs and Example Projects

Goal:
Make the MCP-first story obvious to new users and agents.

Focus:

- Expand `MCP.md` with real examples.
- Add example MCP client configs.
- Add "good agent usage" examples.
- Add "bad agent usage" examples.
- Add one small demo project showing:
  - `session_start`
  - `guidance`
  - `context_pack`
  - `verify`
  - `record`
  - `reconcile`
- Add troubleshooting section.

Success outcome:
A developer can copy the MCP config, connect Haxaml, and understand how an agent should use it within 10 minutes.

## 0.4.4: Map-Aware Verification v1

Goal:
Make verification smarter when `map.yaml` exists.

Focus:

- Tie touched files to mapped modules.
- Detect when a task affects modules that were not planned.
- Require impact evidence for mapped high-risk modules.
- Add verify warnings when module dependencies may be affected.
- Improve `session_record` gate reasons when map evidence is missing.

Success outcome:
If an agent changes `auth`, `payments`, or `inventory`, Haxaml can tell whether the agent acknowledged the correct impact area.

## 0.4.5: Context Pack Compaction

Goal:
Reduce token waste while keeping agents grounded.

Focus:

- Add deterministic context budget levels:
  - `minimal`
  - `standard`
  - `full`
- Include only task-relevant FRAME sections.
- Add metadata showing what was included/excluded.
- Add `omitted_context` notes so agents know what they did not see.
- Improve context-pack output for MCP clients.

Success outcome:
Agents get enough context to work safely without dragging the whole project journal into every task.

## 0.4.6: Dogfooding and Stability

Goal:
Use Haxaml on Haxaml and at least one real project.

Focus:

- Run Haxaml through real agent sessions.
- Capture where agents misuse tools.
- Improve error messages.
- Improve docs based on actual confusion.
- Add regression tests from real bugs.
- Tighten changelog and migration notes.

Success outcome:
The 0.4 line feels stable enough to start preparing the 0.5 direction.
